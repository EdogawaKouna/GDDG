# 🎮 游戏机制文案生成器 - 使用教程

> 版本 1.0 | WPF 桌面应用 | C# .NET 8

---

## 一、环境准备

### 需要安装

| 软件 | 说明 |
|------|------|
| .NET 8 SDK | 运行和编译项目必需 |
| Visual Studio 2022 | 开发/编译（可选，有 .NET 就能跑） |

### 检查是否已安装

打开命令行运行：
```
dotnet --version
```
如果显示 `8.x.x` 说明已安装。如果没有，去 https://dotnet.microsoft.com/download/dotnet/8.0 下载安装。

---

## 二、获取 DeepSeek API Key

本工具通过调用 DeepSeek 的 AI 接口生成策划文档，需要 API Key。

### 获取步骤

1. 打开 https://platform.deepseek.com/
2. 注册/登录账号
3. 进入 API Keys 页面 → 创建新的 API Key
4. 复制 Key（格式：`sk-xxxxxxxxxxxxxxxx`）
5. 充值少量余额（生成一次文档约消耗 0.01-0.02 元，几十元能用很久）

---

## 三、配置与运行

### 方法一：从源码运行

1. 用 Visual Studio 2022 打开项目文件夹
2. 打开 `MainWindow.xaml.cs`
3. 找到第 17 行：
   ```csharp
   private const string ApiKey = "sk-你的DeepSeekKey";
   ```
4. 把 `sk-你的DeepSeekKey` 替换成你的真实 Key
5. 按 `F5` 或点击运行按钮

### 方法二：编译发布为独立程序

在项目目录打开命令行，运行：
```
dotnet publish -c Release -r win-x64 --self-contained true
```
编译完成后，在 `bin\Release\net8.0-windows\win-x64\publish\` 文件夹里找到 `.exe` 文件，可以复制到任何 Windows 电脑上运行（无需安装 .NET SDK）。

## 四、界面功能介绍

### 界面总览

```
┌──────────────────────────────────────────────────────────────┐
│  🎮 游戏机制文案生成器                                       │
├──────────────────────────────────────────────────────────────┤

│  💡 核心特色: [____________]  📱 目标平台: [____________]   │
│  [🚀 生成文档]  状态: 就绪     ███████░░░░░░░░ 进度条       │
├──────────────────────────────────────────────────────────────┤
│  📄 核心玩法规则 | 💰 经济系统 | ⚔️ 战斗/数值 | 📈 成长... │  ← 选项卡
│  ┌────────────────────────────────────────────────────────┐  │
│  │  这里显示对应模块的生成内容...                          │  │
│  │                                                        │  │
│  └────────────────────────────────────────────────────────┘  │
├──────────────────────────────────────────────────────────────┤
│  迭代修改: [下拉选模块]  [输入修改要求...] [🔄 应用修改]   │
│  [📁 导出全部]                                              │
└──────────────────────────────────────────────────────────────┘
```

### 各区域说明

#### 1. 输入区（顶部）
四个输入框分别填写游戏的核心设定信息。

**示例输入：**
| 字段 | 示例内容 |
|------|----------|
| 游戏类型 | 2D roguelike souls-like |
| 背景设定 | 玩家是一只骷髅怪物，夹在勇者和魔王之间 |
| 核心特色 | 可以捡怪物断臂进化升级 |
| 目标平台 | PC |

**注意：** 游戏类型和背景设定为必填项。

#### 2. 生成按钮
点击「🚀 生成文档」开始生成。生成过程中进度条会逐步前进（共 7 步：名称 + 6 个模块）。等待 30-60 秒即可完成。

#### 3. 输出区（中间）
用选项卡分 6 个模块展示生成的策划内容：
- **核心玩法规则** —— 游戏流程、战斗模式、胜利条件、操作映射、核心循环
- **经济系统** —— 资源类型、获取途径、消耗曲线、循环设计
- **战斗/数值框架** —— 属性定义、伤害公式、成长曲线、平衡策略
- **成长系统** —— 等级体系、技能树、解锁节奏、长线养成
- **商业化方案** —— 付费点设计、平衡策略、免费玩家体验
- **竞品分析** —— 类似游戏对比、差异化建议、市场定位

#### 4. 迭代修改区（底部）
可以对某个模块进行针对性修改，而不影响其他模块。

**操作步骤：**
1. 下拉选择要修改的模块（如「战斗/数值框架」）
2. 在输入框写入修改要求（如「改成注重闪避和反击的战斗风格」）
3. 点击「🔄 应用修改」
4. 等待 AI 重新生成该模块内容，自动更新到对应选项卡

#### 5. 导出按钮
点击「📁 导出全部」会生成三个文件到程序运行目录：
- `game_design_doc.md` —— Markdown 格式，适合阅读和分享
- `game_design_doc.json` —— JSON 格式，适合程序解析
- `game_design_doc.txt` —— 纯文本格式，通用性强

## 五、使用流程（完整示例）

### 场景：设计一款克苏鲁题材的肉鸽魂类游戏

**第一步：填写设定**
| 字段 | 内容 |
|------|------|
| 游戏类型 | 2D roguelike souls-like |
| 背景设定 | 玩家是一只骷髅怪物，夹在勇者和魔王之间 |
| 核心特色 | 可以捡怪物断臂进化升级，刺激收集体验 |
| 目标平台 | PC |

**第二步：点击生成**
进度条逐步前进，大约 30 秒后六个选项卡全部填充内容。

**第三步：查看结果**
切换到每个选项卡查看生成的具体策划方案。重点关注：
- 数值是否合理（伤害公式、成长曲线）
- 玩法逻辑是否通顺（核心循环、流程框架）

**第四步：迭代优化（可选）**
比如对战斗系统不满意 → 选「战斗/数值框架」→ 输入「改成注重闪避反击的风格，弹反造成更高伤害」→ 点击应用

**第五步：导出文档**
点击导出，获得标准策划文档，可用于团队协作或进一步编辑。

## 六、常见问题

### Q1：点击生成后没反应/报错
- 检查 API Key 是否填写正确（注意不要有多余空格）
- 检查网络连接是否正常
- 检查 DeepSeek 账户余额是否充足

### Q2：生成内容不理想
- 这是 AI 生成的「第一版草案」，策划需要在此基础上调整
- 使用迭代修改功能针对性优化某个模块
- 可以重新调整输入信息再生成一次

### Q3：生成的内容全是英文
- 检查输入是否使用了中文
- 生成的文本应该是中文，如果出现英文可在迭代中要求「改为中文输出」

### Q4：导出文件在哪？
- 默认在程序运行目录下（如果从 VS 运行，在 `bin\Debug\net8.0-windows\` 下）
- 如果发布了独立 exe，就在 exe 同目录下
